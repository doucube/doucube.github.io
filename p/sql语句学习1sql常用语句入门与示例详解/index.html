一、查询类语句（DQL）

查询类语句的核心目标：从表中读取数据，而不是修改数据。

1. SELECT

作用： 用于查询数据，是 SQL 中最基础、最常用的语句。
常见写法：

1
2

SELECT 列名 FROM 表名;
SELECT * FROM users;

重点：

  • SELECT 表示“选取数据”
  • * 表示查询全部列
  • SELECT 只读取数据，不会修改原表

图表示意：

原表 users

id name age
1 Tom 20
2 Lily 18

执行：

1

SELECT name, age FROM users;

返回结果：

name age
Tom 20
Lily 18

2. DISTINCT

作用： 用于去重，去掉重复值。
示例：

1

SELECT DISTINCT gender FROM users;

重点：

  • DISTINCT 对查询结果去重
  • 它作用于“结果集”，不是删除表中的重复数据

图表示意：

原表 users

id name gender
1 Tom 男
2 Jack 男
3 Lucy 女

执行：

1

SELECT DISTINCT gender FROM users;

返回结果：

gender
男
女

3. WHERE

作用： 用于条件筛选。
示例：

1

SELECT * FROM users WHERE age > 18;

重点：

  • WHERE 放在查询条件中
  • 只有满足条件的数据才会被保留

图表示意：

原表 users

id name age
1 Tom 17
2 Lily 20
3 Jack 22

执行：

1

SELECT * FROM users WHERE age > 18;

返回结果：

id name age
2 Lily 20
3 Jack 22

4. ORDER BY

作用： 用于排序。
示例：

1

SELECT * FROM users ORDER BY age DESC;

重点：

  • ASC 表示升序（默认）
  • DESC 表示降序
  • ORDER BY 通常放在语句靠后位置

图表示意：

原表 users

name age
Tom 20
Lily 18
Jack 25

执行：

1

SELECT * FROM users ORDER BY age DESC;

返回结果：

name age
Jack 25
Tom 20
Lily 18

5. GROUP BY

作用： 用于分组统计。
示例：

1

SELECT dept_id, COUNT(*) FROM employees GROUP BY dept_id;

重点：

  • GROUP BY 会把相同字段值归为一组
  • 常与 COUNT()、SUM()、AVG() 等聚合函数一起使用
  • 分组后，SELECT 中通常只能写分组字段和聚合结果

图表示意：

原表 employees

emp_id name dept_id
1 A 10
2 B 10
3 C 20

执行：

1

SELECT dept_id, COUNT(*) AS total FROM employees GROUP BY dept_id;

返回结果：

dept_id total
10 2
20 1

6. HAVING

作用： 用于对分组后的结果再次筛选。
示例：

1
2
3
4

SELECT dept_id, COUNT(*)
FROM employees
GROUP BY dept_id
HAVING COUNT(*) > 5;

重点：

  • WHERE 是分组前筛选
  • HAVING 是分组后筛选
  • HAVING 经常配合 GROUP BY 使用

图表示意：

分组后的结果

dept_id total
10 3
20 8
30 6

执行：

1
2
3
4

SELECT dept_id, COUNT(*) AS total
FROM employees
GROUP BY dept_id
HAVING COUNT(*) > 5;

返回结果：

dept_id total
20 8
30 6

7. LIMIT / OFFSET

作用： 用于限制返回行数。
示例：

1
2

SELECT * FROM users LIMIT 10;
SELECT * FROM users LIMIT 10 OFFSET 20;

重点：

  • LIMIT 10 表示取前 10 行
  • OFFSET 20 表示跳过前 20 行
  • 常用于分页查询

图表示意：

假设 users 一共有 30 行数据：

操作 含义
LIMIT 10 取第 1~10 行
LIMIT 10 OFFSET 20 跳过前 20 行，取第 21~30 行

8. AS

作用： 用于起别名。
示例：

1

SELECT name AS 名字, age AS 年龄 FROM users;

重点：

  • 可以给列起别名
  • 也可以给表起别名
  • 别名能让查询结果更清晰，更适合展示

图表示意：

执行：

1

SELECT name AS 名字, age AS 年龄 FROM users;

返回结果：

名字 年龄
Tom 20
Lily 18

9. CASE WHEN

作用： 用于条件判断。
示例：

1
2
3
4
5
6

SELECT name,
       CASE
         WHEN age >= 18 THEN 'adult'
         ELSE 'minor'
       END AS status
FROM users;

重点：

  • 类似编程语言中的 if...else
  • 常用于结果分类、评分、状态标记
  • CASE WHEN 会生成一个新的结果列

图表示意：

原表 users

name age
Tom 20
Lily 16

执行后结果：

name status
Tom adult
Lily minor

二、条件运算常用关键字

这一类内容通常与 WHERE 搭配使用，用来控制筛选逻辑。

1. 比较运算符：=、!=、<>、>、<、>=、<=

作用： 用于数值或字符串比较。
示例：

1

SELECT * FROM users WHERE age >= 18;

重点：

  • = 等于
  • != 或 <> 不等于
  • >= 大于等于
  • 比较运算是 WHERE 条件最基础的组成部分

图表示意：

age 是否满足 age >= 18
16 否
18 是
25 是

2. AND / OR / NOT

作用： 用于逻辑运算。
示例：

1

SELECT * FROM users WHERE age > 18 AND gender = 'M';

重点：

  • AND：同时满足
  • OR：满足其一
  • NOT：取反
  • 复杂条件筛选几乎都会用到逻辑运算符

图表示意：

条件 结果
age > 18 AND gender='M' 两个条件都必须为真
age > 18 OR gender='M' 任一条件为真即可
NOT age > 18 对条件结果取反

3. IN / NOT IN

作用： 用于匹配多个固定值。
示例：

1

SELECT * FROM users WHERE id IN (1, 2, 3);

重点：

  • IN 相当于“在这个列表里”
  • NOT IN 相当于“不在这个列表里”
  • 当条件值较多时，比多个 OR 更简洁

图表示意：

id 是否在 (1,2,3) 中
1 是
2 是
4 否

4. BETWEEN … AND …

作用： 用于范围查询。
示例：

1

SELECT * FROM users WHERE age BETWEEN 18 AND 30;

重点：

  • 表示查询某个区间范围
  • 一般包含边界值
  • BETWEEN 18 AND 30 通常等价于 age >= 18 AND age <= 30

图表示意：

age 是否在 18~30 之间
17 否
18 是
25 是
30 是
31 否

5. LIKE / NOT LIKE

作用： 用于模糊匹配。
示例：

1

SELECT * FROM users WHERE name LIKE 'A%';

重点：

  • % 表示任意长度字符
  • _ 表示单个字符
  • LIKE 不是独立语句，而是条件匹配运算符

图表示意：

模式 含义 可匹配示例
'A%' 以 A 开头 Alice
'%A' 以 A 结尾 LunaA
'%A%' 包含 A JackA
'_A%' 第二位是 A MAx

6. IS NULL / IS NOT NULL

作用： 用于判断空值。
示例：

1

SELECT * FROM users WHERE email IS NULL;

重点：

  • NULL 表示“没有值”
  • 判断空值必须使用 IS NULL
  • 不能使用 = NULL 来判断空值

图表示意：

email IS NULL 结果
NULL 是
a@test.com 否

7. EXISTS / NOT EXISTS

作用： 用于判断子查询是否有结果。
示例：

1
2
3
4

SELECT * FROM users u
WHERE EXISTS (
  SELECT 1 FROM orders o WHERE o.user_id = u.id
);

重点：

  • 只要子查询返回至少一行，EXISTS 就成立
  • SELECT 1 中的 1 只是常量，值本身不重要
  • EXISTS 更关注“是否存在”，而不是具体返回什么字段

图表示意：

users 表

id name
1 Tom
2 Lily

orders 表

order_id user_id
1001 1

执行后结果：

id name
1 Tom

说明：只有 Tom 在 orders 中存在对应订单记录。

8. ANY / SOME / ALL

作用： 用于与子查询结果集进行比较。
示例：

1
2
3
4

SELECT * FROM products
WHERE price > ALL (
  SELECT price FROM products WHERE category = 'A'
);

重点：

  • ANY / SOME：满足结果集中任意一个即可
  • ALL：必须满足结果集中的全部值
  • ALL 的限制最严格

图表示意：

A 类产品价格集合：

price
100
120
150

若条件为 price > ALL (...)，则：

price 是否满足
130 否
160 是

三、聚合函数

聚合函数用于“统计整组数据”，常与 GROUP BY 搭配。

1. COUNT()

作用： 统计数量。
示例：

1

SELECT COUNT(*) FROM users;

重点：

  • COUNT(*) 统计总行数
  • 最常用于统计总数、人数、订单数等

图表示意：

users 表共有 3 条记录：

id
1
2
3

执行后结果：

COUNT(*)
3

2. SUM()

作用： 求和。
示例：

1

SELECT SUM(salary) FROM employees;

重点：

  • 适合统计金额、分数、数量等数值字段
  • 只能对数值列求和

图表示意：

name salary
A 3000
B 5000
C 4000

执行后结果：

SUM(salary)
12000

3. AVG()

作用： 求平均值。
示例：

1

SELECT AVG(score) FROM students;

重点：

  • 用于统计平均分、平均工资、平均价格等
  • AVG 通常会自动忽略 NULL

图表示意：

name score
A 80
B 90
C 70

执行后结果：

AVG(score)
80

4. MAX() / MIN()

作用： 用于找最大值和最小值。
示例：

1

SELECT MAX(age), MIN(age) FROM users;

重点：

  • MAX() 求最大值
  • MIN() 求最小值
  • 适用于年龄、价格、日期、分数等字段

图表示意：

name age
Tom 18
Lily 25
Jack 20

执行后结果：

MAX(age) MIN(age)
25 18

四、连接查询（JOIN）

连接查询用于把多张表按某个关联字段拼接到一起。

1. INNER JOIN

作用： 只保留两张表中能匹配上的数据。
示例：

1
2
3

SELECT u.name, o.order_id
FROM users u
INNER JOIN orders o ON u.id = o.user_id;

重点：

  • ON 用于指定连接条件
  • INNER JOIN 只显示双方都有对应关系的数据

图表示意：

users

id name
1 Tom
2 Lily

orders

order_id user_id
1001 1
1002 3

结果：

name order_id
Tom 1001

2. LEFT JOIN

作用： 保留左表全部数据，右表匹配不到则补 NULL。
示例：

1
2
3

SELECT u.name, o.order_id
FROM users u
LEFT JOIN orders o ON u.id = o.user_id;

重点：

  • 左表全部保留
  • 非常适合查“全部用户 + 是否下单”这类场景

图表示意：

结果：

name order_id
Tom 1001
Lily NULL

3. RIGHT JOIN

作用： 保留右表全部数据，左表匹配不到则补 NULL。
示例：

1
2
3

SELECT u.name, o.order_id
FROM users u
RIGHT JOIN orders o ON u.id = o.user_id;

重点：

  • 右表全部保留
  • RIGHT JOIN 与 LEFT JOIN 方向相反

图表示意：

结果：

name order_id
Tom 1001
NULL 1002

4. FULL JOIN / FULL OUTER JOIN

作用： 两边都保留，不匹配的一侧用 NULL 填充。
示例：

1
2
3

SELECT *
FROM a
FULL OUTER JOIN b ON a.id = b.id;

重点：

  • 同时保留左右两表所有记录
  • 适合查找“匹配 + 不匹配”的整体情况

图表示意：

a.id b.id 说明
1 1 双方匹配
2 NULL 仅 A 有
NULL 3 仅 B 有

5. CROSS JOIN

作用： 返回笛卡尔积，即两表每一行互相组合。
示例：

1

SELECT * FROM a CROSS JOIN b;

重点：

  • 若 A 表有 2 行，B 表有 3 行，结果就是 6 行
  • 使用时要谨慎，否则结果会急剧膨胀

图表示意：

a 表：

a
A1
A2

b 表：

b
B1
B2

结果：

a b
A1 B1
A1 B2
A2 B1
A2 B2

6. SELF JOIN

作用： 自连接，即同一张表和自己连接。
示例：

1
2
3

SELECT a.name, b.name
FROM employees a
JOIN employees b ON a.manager_id = b.id;

重点：

  • 常用于处理上下级、父子级、分类树等关系
  • 本质上仍然是 JOIN，只是两边用的是同一张表

图表示意：

employees

id name manager_id
1 Alice NULL
2 Bob 1

结果：

员工 经理
Bob Alice

五、集合操作

集合操作用于把多个查询结果按“并集、交集、差集”的方式处理。

1. UNION

作用： 合并结果并去重。
示例：

1
2
3

SELECT name FROM a
UNION
SELECT name FROM b;

重点：

  • 两边列数要一致
  • 对应列类型要兼容
  • UNION 会自动去除重复行

图表示意：

A 表结果：

name
Tom
Lily

B 表结果：

name
Lily
Jack

执行后结果：

name
Tom
Lily
Jack

2. UNION ALL

作用： 合并结果但不去重。
示例：

1
2
3

SELECT name FROM a
UNION ALL
SELECT name FROM b;

重点：

  • 与 UNION 的区别在于不去重
  • 通常性能比 UNION 更高

图表示意：

执行后结果：

name
Tom
Lily
Lily
Jack

3. INTERSECT

作用： 取交集，即两边都存在的数据。
示例：

1
2
3

SELECT name FROM a
INTERSECT
SELECT name FROM b;

重点：

  • 返回共同拥有的结果
  • 适合找重合数据

图表示意：

结果：

name
Lily

4. EXCEPT / MINUS

作用： 取差集，即在 A 中有、在 B 中没有的数据。
示例：

1
2
3

SELECT name FROM a
EXCEPT
SELECT name FROM b;

重点：

  • EXCEPT 常见于部分数据库
  • Oracle 中常用 MINUS
  • 本质都是差集查询

图表示意：

结果：

name
Tom

六、数据操作类（DML）

DML 用于对表中已有数据进行增删改。

1. INSERT INTO

作用： 插入新数据。
示例：

1

INSERT INTO users(name, age) VALUES ('Tom', 20);

重点：

  • 指定列名时更安全、更清晰
  • INSERT 是“新增一行”最常用的语句

图表示意：

插入前 users

id name age
1 Lily 18

执行插入后：

id name age
1 Lily 18
2 Tom 20

2. DELETE

作用： 删除数据。
示例：

1

DELETE FROM users WHERE id = 1;

重点：

  • DELETE 删除满足条件的记录
  • 如果不写 WHERE，可能删除整张表的数据

图表示意：

删除前：

id name
1 Tom
2 Lily

执行：

1

DELETE FROM users WHERE id = 1;

删除后：

id name
2 Lily

总结

  • 查询数据：SELECT、WHERE、ORDER BY、GROUP BY
  • 处理条件：IN、BETWEEN、LIKE、EXISTS
  • 做统计：COUNT()、SUM()、AVG()、MAX()、MIN()
  • 拼接多表：INNER JOIN、LEFT JOIN、RIGHT JOIN
  • 合并结果：UNION、INTERSECT、EXCEPT
  • 操作数据：INSERT、DELETE
