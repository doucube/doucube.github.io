摘要

这篇文章是 SQL 入门阶段的第一篇学习笔记，重点解决两个最基础的问题：

  1. 认识数据库、表、字段、行分别是什么。
  2. 理解 Web 请求为什么会连到 SQL。

如果把 SQL 学习比作搭房子，那么这篇内容就是最底层的地基。先把概念认清，再学查询、增删改查和项目实战，会顺很多。

一、SQL 学习中最基础的两个任务

在学习 SQL 时，最基础的任务有两个：

  1. 认识数据库、表、行、字段。
  2. 理解 Web 请求为什么会连到 SQL。

这两个知识点，是后续学习 查询语句、增删改查、表结构设计、项目实战 的基础。如果前面这些概念没有建立起来，后面的 SELECT、INSERT、UPDATE、DELETE 都会学得很乱。

图表理解

基础任务 作用 为什么重要
认识数据库结构 搞清楚数据存在哪里、怎么组织 这是理解 SQL 操作对象的前提
理解 Web 与 SQL 的关系 搞清楚请求为什么会访问数据库 这是理解真实项目运行流程的前提

二、认识数据库、表、行、字段

1. 数据库（Database）

数据库可以理解成一个仓库，专门用来存储数据。

常见的数据库有：

  • MySQL
  • PostgreSQL
  • SQL Server

记忆方式

  • 数据库 = 大仓库
  • 一个数据库里可以放很多张表

图表理解

概念 类比 作用
数据库 大仓库 负责集中存放和管理数据

数据库（Database）
├── users 表
├── orders 表
└── products 表

理解重点： 数据库不是一条数据，也不是一张表，而是“装很多表的整体容器”。

2. 表（Table）

表可以理解成仓库中的货架，每个货架负责存放一类数据，也就是对数据进行分类。

例如：

  • 用户表 users
  • 订单表 orders
  • 商品表 products

记忆方式

  • 表 = 仓库中的货架
  • 一张表通常用来保存同一种类型的数据

图表理解

表名 存放内容 示例
users 用户数据 用户名、年龄、邮箱
orders 订单数据 订单编号、金额、下单时间
products 商品数据 商品名、价格、库存

理解重点： 表的作用是“分类存储数据”，不是把所有数据混在一起。

3. 字段（Field）

字段就是表中的列，表示这类数据有哪些属性。

例如，在用户表中，常见字段可能有：

  • id：用户编号
  • name：用户名
  • age：年龄
  • email：邮箱

记忆方式

  • 字段 = 表头 = 一列数据的名称
  • 字段描述的是“这条数据有哪些属性”

图表理解

字段名 含义 数据类型示意
id 用户编号 整数
name 用户名 字符串
age 年龄 整数
email 邮箱 字符串

理解重点： 字段描述的是“属性”，也就是一条记录应该有哪些信息。

4. 行（Row）

行就是表中的一条具体记录。

例如，在用户表中，一行数据可能是：

id name age email
1 张三 20 a@test.com

这整一行就表示一个具体用户的信息。

记忆方式

  • 行 = 一条完整的数据记录
  • 一行通常对应一个真实对象，比如一个用户、一张订单、一个商品

图表理解

行的含义 对应真实对象
一条用户记录 一个真实用户
一条订单记录 一笔真实订单
一条商品记录 一个真实商品 
1
2

users 表中的一行记录
(1, '张三', 20, 'a@test.com')

理解重点： 字段是“列名”，行是“具体数据”。这一点必须分清。

5. 它们之间的包含关系

它们之间的关系是：

  • 数据库 包含多个 表
  • 表 由 字段 和 行 组成
  • 字段 是列
  • 行 是记录

三、Web 请求为什么会连到 SQL

很多初学者会疑惑：我明明是在浏览器里点按钮，为什么最后会查到数据库？

原因是：浏览器并不会直接操作数据库，而是会先把请求发给后端程序，再由后端程序去执行 SQL。

一个非常常见的流程如下：

  1. 用户在浏览器中发起请求。
  2. 请求到达后端服务。
  3. 后端根据业务逻辑拼装或调用 SQL。
  4. 数据库执行 SQL。
  5. 数据库把结果返回给后端。
  6. 后端再把结果返回给浏览器。

理解重点： 真正连接 SQL 的通常不是浏览器，而是后端程序。

四、一个最简单的完整流程示意

假设用户访问“查看所有用户”页面，背后可能发生的是：

第一步：浏览器发起请求 

1

GET /users

第二步：后端收到请求并准备查询

后端可能会执行这样一条 SQL：

1

SELECT * FROM users;

第三步：数据库返回结果

id name age email
1 张三 20 a@test.com
2 李四 22 b@test.com

第四步：后端把结果返回给浏览器

浏览器最终看到的是用户列表页面，或者一个 JSON 接口响应。

图表理解

步骤 输入 输出
浏览器请求 /users 把请求发给后端
后端查询 SELECT * FROM users; 从数据库取数据
数据库返回 数据表结果 把结果返回后端
页面展示 后端响应 浏览器展示数据

五、学习这一部分时最该记住的内容

如果你是 SQL 初学者，这一部分最应该记住的是下面这几句话：

  1. 数据库是装表的地方。
  2. 表是分类存放数据的结构。
  3. 字段是列，是属性。
  4. 行是记录，是具体数据。
  5. Web 请求一般不会直接访问数据库，而是通过后端程序去执行 SQL。

速记图 

1

浏览器 → 后端 → SQL → 数据库 → 后端 → 浏览器

容易混淆的概念 正确认识

字段 和 行 字段是列名，行是一条记录
浏览器 和 数据库 浏览器不直接连数据库，通常通过后端访问
